Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

Apa yang dimaksud dengan constructor overloading dalam pemrograman berorientasi objek?
Bagaimana cara Java membedakan constructor yang memiliki nama sama namun parameter berbeda?
Apa perbedaan antara mengakses variabel secara langsung (misalnya: objek.variabel = nilai) dengan menggunakan method setter?
Jelaskan konsep encapsulation dan bagaimana method dapat membantu melindungi data dalam sebuah class!
Mengapa constructor tidak memiliki return type, bahkan tidak menggunakan kata kunci void?

Jawaban
    1. Constructor overloading adalah penggunaan beberapa constructor dengan nama sama tetapi parameter berbeda dalam satu kelas.
    2. Java membedakan constructor berdasarkan jumlah, tipe, dan urutan parameter (method signature).
    3. Mengakses variabel langsung tidak aman, sedangkan menggunakan setter memberi validasi dan perlindungan data.
    4. Encapsulation menyembunyikan data dengan membuat variabel private dan mengaksesnya melalui method untuk menjaga keamanan data.
    5. Constructor tidak memiliki return type karena tugasnya langsung menginisialisasi objek, bukan mengembalikan nilai
